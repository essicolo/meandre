# slso-od case

SLSO basin built entirely from open data (Copernicus DEM + ESA WorldCover +
SoilGrids + JRC water + OSM rivers/lakes + Open-Meteo ERA5 forcing + HYDAT
discharge observations). Same bbox and temporal split as the legacy
`notebooks/slso/` case for direct comparison.

## Workflow

```powershell
# 1. Build basin + forcing + observations (~15 min first time, then cached)
python scripts/bbox_to_basin.py `
    --bbox=-73.0,44.5,-69.6,47.7 `
    --output=.models/slso-od/data/basin.duckdb `
    --min-area-km2=1.5 --max-subcatchments=3500 `
    --with-forcing --start=2000-01-01 --end=2024-12-31 `
    --with-observations

# 2. Train (uses the existing slso.py + the new config)
python notebooks/slso/slso.py .models/slso-od/config/slso-od.toml
```

## Layout

```
.models/slso-od/
  config/
    slso-od.toml          # training config (paths point under this dir)
  data/
    basin.duckdb          # generated by bbox_to_basin
    forcing.nc            # generated by bbox_to_basin
    geo_cache/            # rasters source (DEM, LC, etc.) — re-creatable
  checkpoints/
    best.pt               # populated by training
  results/
    fields.nc, reach.parquet
  runs.duckdb             # mlflow-style run log
```

The whole `.models/` dir is gitignored — local artefacts only.

## Notes

- Discharge observations come from HYDAT (ECCC), filtered to bbox at build time.
  The HYDAT SQLite (~1.3 GB) is cached in `.models/_shared/hydat/` and shared
  across all Canada cases.
- Withdrawals are NOT auto-fetched. To populate the `withdrawals` table, use
  `scripts/ingest_withdrawals.py` with a manually curated parquet (e.g.
  `io-eau-meandre.parquet`).
- The `weather_grid` path in the config points to `forcing.nc` directly
  (legacy field, the file is what's actually read by `extract_forcing()`).
